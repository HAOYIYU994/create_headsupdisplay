# Create Heads Up Display - MC百科模组介绍

## 模组简介

**Create Heads Up Display（简称 HUD）** 是一个基于机械动力（Create）的 NeoForge 扩展模组，为 Minecraft 添加了一套完整的**穿戴式 AR 抬头显示系统**与**世界显示屏面板**。玩家可以通过数据链路将各种信号源（红石信号、NBT 探针、雷达、物理引擎等）汇集、处理、排版，最终以 12 种仪表样式渲染在头盔 HUD 或世界中的大屏幕上。

---

## 核心设备

### 1. 头戴显示器（Head Mount Display）
可穿戴头盔。装备后会在视野上方叠加 HUD 界面，显示来自绑定的显示终端的自定义数据。潜行右键绑定终端。

### 2. 显示终端（Display Terminal）
基础版终端方块。右键打开配置界面，可拖拽布置数据槽位、静态文字、图片和雷达元素，调整位置、缩放、旋转、颜色和透明度。支持 12 种显示模式。

### 3. 显示终端·Pro（Display Terminal Pro）
进阶版终端，新增：
- **图层系统**：支持多层叠加管理，每层可独立控制可见、冻结、锁定
- **源卡坞**：将数据源拖拽为卡片，一个槽位可绑定多个数据源（用于 XYZ、陀螺仪等需要多维数据的模式）
- **动画编辑器**：基于时间轴的关键帧动画，支持触发条件（始终/高于/低于/区间）、循环、结束行为（停止/完成/保持/反转）
- **画布缩放**

### 4. 数据集成核心（OmniCore）
系统的中央数据枢纽。接收 6 种数据源：
- **红石信号**：通过频率物品绑定红石信号强度
- **Display Link**：接收机械动力 Display Link 推送的数据
- **图片**：上传自定义图片（需安装 Image 插件）
- **雷达**：接入 Create Radar 模组的雷达数据（需 Radar 插件）
- **探针**：通过 NBT Reader + Plugin Base 读取任意方块的 NBT/属性数据
- **物理引擎**：读取 Sable 等模组的物理朝向数据

内置 27 格插件栏、自动排序、转译表达式引擎等。

### 5. 链路方块（Link Block）
小型贴面方块，用于连接 OmniCore 与显示终端或雷达 Monitor。右键 OmniCore 记录位置，再右键终端自动放置并绑定。破坏后自动解绑。

### 6. 显示面板（Display）⚡
世界中的多向方块显示屏。支持 1×1 到 32×16 的矩形组合，拥有 10 种形状变体。放置后自动绑定附近终端，将 HUD 内容投射到世界中。

### 7. NBT 读取器（NBT Reader）
贴面探针方块，可读取相邻方块的 NBT 数据。配置 NBT 路径后，将数据推送到与其 Plugin Base 绑定的 OmniCore。支持可配置轮询间隔（1-100 tick）。

### 8. 快速 Display Link（Fast Display Link）
机械动力 Display Link 的高频变体，刷新间隔大幅缩短，实现近乎实时的数据推送。

### 9. 插件物品
- **Image Plugin**：安装到 OmniCore 解锁图片上传功能
- **Radar Plugin**：安装到 OmniCore 解锁雷达接入功能
- **Plugin Base**：基础探针底板，右键绑定 NBT Reader 后插入 OmniCore 即可创建探针数据源

## 显示样式（12 种仪表模式）

| 样式 | 说明 |
|------|------|
| **Text** | 纯文本显示 |
| **Bar** | 可自定义材质纹理的进度条 |
| **Altimeter** | 垂直高度计，带三角指示器 |
| **Dial** | 圆形指针表盘 |
| **Digital** | 数码管 7 段式数字显示 |
| **HUD Alt** | HUD 风格高度指示器 |
| **Compass** | 罗盘，带北向指示 |
| **Speedometer** | 速度表，可切换单位 |
| **Chart** | 滚动折线图，带历史缓存 |
| **XYZ** | 三轴坐标显示（X/Y/Z） |
| **LED Array** | LED 点阵面板，可配置行列数与颜色 |
| **Gyro** | 陀螺仪/人工地平仪（俯仰 + 滚转），需 2 个数据源 |
| **外部加载** | 支持从 `create_headsupdisplay_modes/` 文件夹加载自定义仪表 .class 文件 |

## 转译系统

支持 4 种信号值转换模式：
1. **无（None）**：原始值直出
2. **表达式（Expression）**：数学表达式求值，支持 `+`、`-`、`*`、`/`、`^`、括号，变量 `x` 代表原始值
3. **条件判断（Conditional）**：If/ElseIf/Else 条件链，支持 6 种运算符（`>`、`>=`、`==`、`<=`、`<`、`!=`），可用 `${name}` 引用其他源的值
4. **图片条件（Image Conditional）**：根据信号值自动切换显示的图片
5. **正则提取**：所有模式均支持从文本中提取数字作为输入

---

## 动画系统（Pro 版专属）

- 基于时间轴的关键帧编辑器
- 每帧可设定：时间、X/Y 位置、缩放、旋转、颜色、透明度
- 触发条件：始终播放 / 高于阈值 / 低于阈值 / 区间内
- 循环与结束行为：停止 / 完成动画 / 保持末帧 / 反转播放

---

## HUD 渲染特性

- **世界锁定模式**：HUD 元素可冻结在终端所在的世界坐标，移动视角时产生 AR 视差效果
- **雷达叠加**：迷你地图式雷达显示，包含扫掠线、玩家标记、实体分类图标（玩家绿/敌对黄/BOSS红/弹射物橙/载具蓝/被动灰）
- **图片显示**：支持上传 PNG 图片叠加显示
- **动态纹理缓存**：异步加载与缓存机制

---

## 联动与集成

| 模组 | 联动内容 |
|------|----------|
| **Create（机械动力）** | 核心依赖。Display Link 数据推送、Display Target 注册、Goggle 头盔判定 |
| **Create Radar** | 可选软依赖（≥0.4）。雷达 Monitor/Bearing 接入 OmniCore，HUD 雷达显示 |
| **Sable / Aeronautics** | 通过反射读取物理引擎朝向，在移动载具上维持 HUD 世界锁定 |

## Ponder 教程

模组内置 2 个 Ponder 场景：
1. **数据集成核心**：8 步完整教程，讲解核心功能、连接 Display Link、放置终端、绑定链路
2. **显示终端**：5 步教程，讲解数据转图像、连接显示面板、装备头戴显示器

## 使用流程简图

```
[红石信号 / Display Link / NBT Reader / 图片 / 雷达]
        ↓
[数据集成核心 OmniCore] ← 链路方块 → [显示终端 / 显示终端 Pro]
        ↓                              ↓
[转译处理 + 自动排序]          [HUD 布局排版 + 动画]
        ↓                              ↓
                              [头戴显示器 HUD]
                              [显示面板 (世界屏幕)]
```

---

## 注意事项

- 本模组为 NeoForge 平台（1.21.1），不兼容 Forge/Fabric
- 机械动力（Create）为必需前置
- Create Radar 为可选依赖，无此模组时雷达功能自动禁用
- 图片上传大小限制可在服务端配置（默认 512 KB）
- HUD 最大缩放可在服务端配置（默认 5 倍）
